javaScript runs in the browser by the help of a special engines which converts our js files into machine code. depending on browsers this engines varies for chrome this is done by V8 Engine. Butt all follow EcmaScript standard.

Breif of Working Process:
(i) At first the code is parsed line by line by parser of V8 Engines . Here all the codes is read and if any syntactical error fount it will return and error message in the console.
(ii) Then a tree like structure is produced which is called Abstract Sytax Tree (AST) which represent all the operation in a tree like structure and Machine code is generated by the help of AST
(iii) After that our system processors understands the instructions and runs in the browser.

So the things happens here in short is the code sent to browser it is transfered to These engines and then these engines compile these into machine language then our Computer understands and give output

Local Storage and Session Storage both are Web Storage objects that save our experience of browsing in JSON string format as key-value pairs.These Web Storage objects can't be sent to the serve site and can only be edited by different javaScript methods.

Differences :
(i)Local Storage stores data for permanent but Session Storage stores for only one session
(ii)Local Storage is accessible from any window but Session Storage is accessible from one tab close
(iii)In terms of capacity Local Storage allows 10mb but Session Storage allows 5mb
(iv)Local storage is not session based but Session Storage is session based
(v)For Session Storage Returning or closing of tab deletes all records but for Local Storage it need to delete manually or via JS methods.

Some methods for Local Storage and Session Storage : setItem(key, value) , getItem(key) , removeItem(key) , clear() , length etc ...
